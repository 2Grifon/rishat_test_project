## Развёртка

### Клонирование репозитория
```
git clone git@github.com:2Grifon/rishat_test_project.git
cd rishat_test_project
```

### Запуск проекта локально
1. Убедиться что в проекте присутствует корректный **.env** файл с необходимыми настройками. В случае отсутствия создать на основе шаблона **default.env**
```
cp default.env .env
```
2. Перейти в каталог с [makefile](Makefile)
3. Использовать команду `make rebuild-loc` для сборки проекта и запуска контейнеров
```
make rebuild-loc
```
4. Испрользовать команду `make migrate` для применения миграций и создания БД
```
make migrate
```
5. Собрать статику для панели администратора. Статика без nginx нормально отдаётся только при debug=true
```
make app-bash
python app/manage.py collectstatic
```
6. Создать пользователя-администратора
```
make app-bash
python app/manage.py createsuperuser
```
7. Приложение будет доступно на http://localhost:8000/api/v1/, панель администратора на http://localhost:8000/admin/

## Настройки, управление проектом
### Настройки .env файла
- **Пример пустого .env файла**
 [default.env](default.env)

- **Django**
  - `SECRET_KEY` - Секретный ключ django
  - `DEBUG` - Режим отладки (true/false)

- **Stripe**
  - `STRIPE_SECRET_KEY` - Секретный ключ Stripe
  - `STRIPE_PUBLISHABLE_KEY` - Публичный ключ Stripe для веб страниц

- **Postgres** - Все настройки этого пункта будут использованы при первом создании БД, не рекомендуется изменять после создания базы
  - `POSTGRES_DB` - Имя базы данных на сервере, в которой будут храниться все табилцы созданные по моделям джанго. Пример: `postgres`
  - `POSTGRES_USER` - Имя пользователя с доступом к БД из пункта `POSTGRES_DB`. Пример: `postgres`
  - `POSTGRES_PASSWORD` - Пароль от пользователя, можно использовать любой безопасный случайный пароль
  - `POSTGRES_HOST` - Название сервера базы данных. По умолчанию должен быть: `postgres`

### Команды make
 - **build** - сборка образа проекта
 - **rebuild** - пересборка проекта с пересозданием контейнеров
 - **rebuild-loc** - пересборка проекта локально с использованием docker-compose.local.yml
 - **up** - запуск проекта
 - **up-loc** - запуск проекта локально
 - **start** - запуск остановленных контейнеров
 - **stop** - остановка контейнеров без удаления
 - **restart** - перезагрузка контейнеров
 - **down** - остановка и удаление контейнеров
 - **reup** - остановка и запуск проекта заново
 - **reup-loc** - остановка и локальный запуск проекта
 - **destroy** - полное удаление контейнеров и образов
 - **logs** - просмотр логов контейнеров
 - **app-logs** - просмотр логов backend
 - **app-bash** - вход в bash контейнер backend
 - **db-bash** - вход в bash контейнер postgres
 - **migrations** - создание миграций
 - **migrate** - применение миграций к БД
 - **psql** - вход в PostgreSQL консоль
 - **install-req** - установка зависимостей
 - **install-local** - установка зависимостей для локальной разработки

### Файловая структура
- **backend/** - основной сервис Python
  - **app/** - основное приложение Django
    - **apps/shop/** - приложение магазина
      - **admin/** - конфигурация Django админ-панели
      - **migrations/** - миграции БД
      - **models/** - моделей данных
      - **serializers/** - сериализаторы для DRF
      - **services/** - бизнес-логика
        - **api_managers/** - менеджеры для внешних API (Stripe)
      - **templates/shop/** - HTML шаблоны
      - **views/** - представления
    - **config/** - конфигурация проекта Django
    - **manage.py** - утилита управления Django
  - **media/** - загруженные файлы пользователей
  - **Dockerfile** - конфигурация Docker контейнера
  - **pyproject.toml** - зависимости Python проекта
- **nginx/** - конфигурация nginx
  - **conf/nginx.conf** - файл конфигурации nginx
- **docker-compose.yml** - конфиг сервисов для продакшена
- **docker-compose.local.yml** - конфиг сервисов для локальной разработки
- **Makefile** - часто используемые команды
- **default.env** - пример переменных окружения
- **README.md** - данная документация
